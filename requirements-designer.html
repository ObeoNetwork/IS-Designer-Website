{% include block_text_image.html txt_order="order-1" title="Saisir les exigences" desc="

Les exigences fonctionnelles définissent les fonctionnalités attendues d’éléments du système. Avec Requirement Designer, vous pouvez attacher des exigences fonctionnelles à n’importe quel élément que vous avez modélisé, de manière transverse dans tous les modeleurs disponibles d’IS Designer.

Vous pouvez également définir des exigences non fonctionnelles. Les exigences sont identifiables, et peuvent être catégorisées en fonction des besoins.

" lnk_txt="" lnk_href="" lnk_target="" lnk_icon="" img_order="order-2" img_src="images/screenshots/requirements/exigences.png" img_title="Saisie d'exigences" %}
{% include block_text_image.html txt_order="order-2" title="Etablir la traçabilité avec les composants de l’application" desc="

Lorsque vous attachez une exigence, fonctionnelle ou non, à un élément modélisé dans IS Designer, Requirement Designer modifie sa représentation afin d’indiquer qu’une exigence lui est attaché.

Vous pouvez ainsi facilement retrouver les exigences définies à partir de n’importe quel modeleur d’IS Designer.
