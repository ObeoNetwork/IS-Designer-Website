{% include block_text_image.html txt_order="order-1" title="Modéliser les use-cases et les acteurs" desc="

Une application permet à des utilisateurs de dérouler des scénarios d’usage pour réaliser des objectifs métier.

Avec Graal Designer, définissez facilement ces scénarios sous la forme de cas d’usage (use-case) et de profils d’utilisateurs (acteurs) qui les réalisent.

Ces cas d’usage peuvent être définis de manière hiérarchique pour préciser les différents scénarios. De même pour les acteurs afin d’identifier des profils différents.

{% include block_text_image.html txt_order="order-2" title="Décomposer les use-cases en tâches" desc="

Les cas d’usage peuvent être décomposés en ensembles de tâches. Ces tâches peuvent être ordonnées et hiérarchisées afin d’établir des scénarios d’utilisations de manière exhaustive.

Vous pouvez associer vos acteurs à des sous-tâches particulières, afin de représenter leurs implications durant les différentes étapes de vos processus métier.

Graal Designer permet d’intégrer un ensemble de tâches à des scénarios d’usage existants, de les partager entre plusieurs scénarios, et de naviguer entre-elles avec simplicité.

" lnk_txt="" lnk_href="" lnk_target="" lnk_icon="" img_order="order-1" img_src="images/screenshots/graal/taches.png" img_title="Diagramme de use-cases" %}
{% include block_text_image.html txt_order="order-1" title="Modéliser les interactions système/utilisateur" desc="

Graal Designer permet la représentation des échanges entre les utilisateurs et les différents composants du système. Vous pouvez ainsi représenter les messages transitants dans le systèmes, entre ses composants mais aussi entre ses acteurs.

Des scénarios d’utilisations avancés prenant en compte parallélisme, synchronisation, ainsi que la création et la destruction d’entités du système peuvent être spécifiés.

Graal Designer peut ainsi être associé aux autres designers présents dans le studio, permettant ainsi une liaison entre les différentes couches de votre application.

" lnk_txt="" lnk_href="" lnk_target="" lnk_icon="" img_order="order-2" img_src="images/screenshots/graal/interaction.png" img_title="Diagramme d'interactions" %}
{% include block_text_image.html txt_order="order-2" title="Modéliser les user-stories" desc="

Les tâches définies avec Graal Designer peuvent être intégrées à des scénarios d’usage plus vastes, prenant aussi en compte les actions des utilisateurs, l’affichage d’écrans, ainsi que les actions devant être effectuées par votre système.

Les scénarios utilisateurs et les tâches définies dans Graal Designer peuvent être intégrées aux user-stories. Vous pouvez spécifier dans quelles conditions effectuer vos sous-tâches, en fonction de l’état du système et des actions de l’utilisateur.
